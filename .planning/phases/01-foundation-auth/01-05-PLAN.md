`async def get_redis(settings: Settings = Depends(get_settings)) -> aioredis.Redis`:
  - Returns `aioredis.from_url(settings.REDIS_URL, decode_responses=True)`

`async def get_current_user(credentials: HTTPAuthorizationCredentials = Depends(bearer_scheme), session: AsyncSession = Depends(get_session), settings: Settings = Depends(get_settings)) -> User`:
  - Decode token: `try: payload = decode_access_token(credentials.credentials, settings.JWT_SECRET_KEY, settings.JWT_ALGORITHM); except JWTError: raise HTTPException(status_code=HTTP_401_UNAUTHORIZED, detail="Invalid or expired token")`
  - Extract: `user_id = payload.get("sub"); jti = payload.get("jti")`
  - If not user_id or not jti: raise HTTPException(status_code=HTTP_401_UNAUTHORIZED, detail="Invalid token claims")
  - Check blocklist: `redis = await get_redis(settings)` then `if await is_token_revoked(jti, redis): raise HTTPException(status_code=HTTP_401_UNAUTHORIZED, detail="Token has been revoked")`
  - Lookup user: `result = await session.execute(select(User).where(User.id == user_id))`; `user = result.scalar_one_or_none()`
  - If not user: raise HTTPException(status_code=HTTP_401_UNAUTHORIZED, detail="User not found")
  - Return user

Router: `router = APIRouter(prefix="/auth", tags=["auth"])`

Endpoints:

`@router.post("/register", response_model=TokenResponse)`:
```
async def register(body: RegisterRequest, session: AsyncSession = Depends(get_session), settings: Settings = Depends(get_settings)) -> TokenResponse:
    token, jti, exp = await register_user(body.email, body.password, session, settings)
    return TokenResponse(access_token=token)
```

`@router.post("/login", response_model=TokenResponse)`:
```
async def login(body: LoginRequest, session: AsyncSession = Depends(get_session), settings: Settings = Depends(get_settings)) -> TokenResponse:
    token, jti, exp = await login_user(body.email, body.password, session, settings)
    return TokenResponse(access_token=token)
```

`@router.post("/logout", status_code=200)`:
```
async def logout(credentials: HTTPAuthorizationCredentials = Depends(bearer_scheme), settings: Settings = Depends(get_settings), redis: aioredis.Redis = Depends(get_redis), current_user: User = Depends(get_current_user)) -> dict:
    payload = decode_access_token(credentials.credentials, settings.JWT_SECRET_KEY, settings.JWT_ALGORITHM)
    await logout_user(payload["jti"], payload["exp"], redis)
    return {"detail": "Logged out"}
```
Note: `current_user` dependency validates the token BEFORE we blocklist it; decode_access_token is called again to extract jti/exp (the token is already validated so this is safe).

`@router.get("/me", response_model=UserOut)`:
```
async def me(current_user: User = Depends(get_current_user)) -> User:
    return current_user
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client→register endpoint | Untrusted email/password arrive; response must not reflect back any credential data |
| HTTP client→protected routes | Bearer token in Authorization header; must be validated before any user-specific data is returned |
| Bearer token→get_current_user | Decoded token's sub/jti claims are used for DB lookup; claims must be treated as untrusted until signature verified |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-05-01 | Information Disclosure | POST /register response | critical | mitigate | response_model=TokenResponse returns ONLY access_token and token_type; password and password_hash fields are absent from TokenResponse model; FastAPI serialises via response_model, excluding unlisted fields |
| T-01-05-02 | Spoofing | GET /me, POST /logout | critical | mitigate | Both routes require Depends(get_current_user) which validates JWT signature, checks Redis blocklist, and verifies user exists in DB |
| T-01-05-03 | Repudiation | POST /logout | critical | mitigate | get_current_user runs before logout_user; if Redis is down, logout_user raises 503 (per D-18); token is never "silently" left active |
| T-01-05-04 | Tampering | Bearer token algorithm | critical | mitigate | decode_access_token in dependencies.py passes algorithms=["HS256"] list — Python-jose rejects tokens with alg!=HS256 including unsigned tokens |
| T-01-05-05 | Elevation of Privilege | SQL injection via email | high | mitigate | SQLAlchemy ORM select(User).where(User.id == user_id) uses parameterized queries; no raw SQL string interpolation |
  </threat_model>
