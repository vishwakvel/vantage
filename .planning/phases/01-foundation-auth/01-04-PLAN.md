Create `app/core/security.py`:

Imports:
- `import uuid` from stdlib
- `from datetime import datetime, timezone`
- `from passlib.context import CryptContext`
- `from jose import JWTError, jwt`

Module-level:
- `pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")`

Functions:

`def hash_password(password: str) -> str`:
  - Returns `pwd_context.hash(password)`

`def verify_password(plain_password: str, hashed_password: str) -> bool`:
  - Returns `pwd_context.verify(plain_password, hashed_password)`

`def create_access_token(user_id: str, expires_seconds: int, secret_key: str, algorithm: str) -> tuple[str, str, int]`:
  - Generates `jti = str(uuid.uuid4())` — unique per token, used for Redis blocklist key
  - Gets current UTC timestamp: `iat = int(datetime.now(timezone.utc).timestamp())`
  - Sets `exp = iat + expires_seconds`
  - payload dict: `{"sub": user_id, "jti": jti, "iat": iat, "exp": exp}`
  - Encodes: `token = jwt.encode(payload, secret_key, algorithm=algorithm)`
  - Returns `(token, jti, exp)` — tuple, not a dict

`def decode_access_token(token: str, secret_key: str, algorithm: str) -> dict`:
  - Returns `jwt.decode(token, secret_key, algorithms=[algorithm])` — IMPORTANT: `algorithms` is a list; this rejects the "none" algorithm because "none" is not in the list
  - Raises `JWTError` on expired, tampered, or invalid token (caller handles)
  </action>

Functions:

`async def register_user(email: str, password: str, session: AsyncSession, settings: Settings) -> tuple[str, str, int]`:
- Normalise email to lowercase: `email = email.lower()`
- SELECT User WHERE User.email == email (case-sensitive after normalise)
- If user found: raise HTTPException(status_code=HTTP_409_CONFLICT, detail="Email already registered")
- Create User(email=email, password_hash=hash_password(password))
- session.add(user); await session.commit(); await session.refresh(user)
- Call create_access_token(str(user.id), settings.JWT_ACCESS_TOKEN_EXPIRE_SECONDS, settings.JWT_SECRET_KEY, settings.JWT_ALGORITHM)
- Return (token, jti, exp)

`async def login_user(email: str, password: str, session: AsyncSession, settings: Settings) -> tuple[str, str, int]`:
- SELECT User WHERE User.email == email.lower()
- If not found or not verify_password(password, user.password_hash): raise HTTPException(status_code=HTTP_401_UNAUTHORIZED, detail="Invalid credentials")
- IMPORTANT: use a single constant-time message for both "user not found" and "wrong password" — never reveal which condition failed
- Call create_access_token and return (token, jti, exp)

`async def logout_user(jti: str, exp: int, redis_client: aioredis.Redis) -> None`:
- `remaining_ttl = max(1, exp - int(time.time()))` — minimum 1 second (Redis requires positive TTL)
- Wrap in try/except Exception:
  - try: `await redis_client.set(f"revoked:{jti}", "", ex=remaining_ttl)`
  - except Exception: raise HTTPException(status_code=HTTP_503_SERVICE_UNAVAILABLE, detail="Token revocation service unavailable")
- Note: remaining_ttl uses max(1, ...) not max(0, ...) because Redis SET with ex=0 is an error

`async def is_token_revoked(jti: str, redis_client: aioredis.Redis) -> bool`:
- Returns `await redis_client.exists(f"revoked:{jti}") > 0`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→register_user | User-supplied email/password arrives; password must never be stored in plaintext |
| client→login_user | User-supplied credentials; timing attacks possible if error messages differ between "no user" and "wrong password" |
| JWT→Redis | JTI in JWT is a UUID; Redis blocklist key must not be forgeable by manipulating JTI |
| logout→Redis | Redis unavailability would leave the token active; must surface as 503 not 200 |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-04-01 | Spoofing | app/core/security.py hash_password | critical | mitigate | passlib CryptContext(schemes=["bcrypt"]) hashes with bcrypt; plaintext password never stored; verify_password uses constant-time comparison internally |
| T-01-04-02 | Tampering | app/core/security.py decode_access_token | critical | mitigate | jwt.decode called with algorithms=["HS256"] (list) — rejects unsigned or "none"-algorithm tokens; signature verification is mandatory |
| T-01-04-03 | Repudiation | app/services/auth_service.py logout_user | critical | mitigate | Redis failure raises HTTP 503 (not 200) per D-18 — caller cannot achieve a "silent logout" that leaves the token active |
| T-01-04-04 | Information Disclosure | app/services/auth_service.py login_user | medium | mitigate | "Invalid credentials" message is identical for missing user and wrong password — prevents username enumeration via differential error messages |
| T-01-04-05 | Tampering | app/services/auth_service.py logout_user TTL | high | mitigate | TTL = max(1, exp - int(time.time())) — always >= token's remaining lifetime; never shorter; prevents TTL-expiry-before-token-expiry attack |
  </threat_model>
